### 2.10. Раздел 8.4: Порядок метеорологического обеспечения

**Требуемые изменения:**

**Полностью переработать с учётом использования MIRA:**

```
8.4.1 Порядок метеорологического обеспечения с использованием системы MIRA

8.4.1.1 Этап планирования (за 24-72 часа до полёта):
а) Оператор БВС получает прогноз погоды через систему MIRA:
   - вводит координаты района работ (или использует геолокацию);
   - строит предполагаемый маршрут полёта;
   - получает прогноз на дату и время планируемого полёта;
   - изучает Risk Score и рекомендации системы.

б) Руководитель полётов анализирует предоставленную информацию:
   - сравнивает прогнозируемые условия с МДМУ;
   - принимает предварительное решение о возможности полёта;
   - при необходимости корректирует дату/время полёта.

в) Результаты этапа фиксируются в Журнале планирования полётов БВС.

8.4.1.2 Этап предполётной подготовки (за 2-4 часа до полёта):
а) Оператор БВС обновляет прогноз в системе MIRA:
   - проверяет актуальность данных (не старше 1 часа);
   - выполняет анализ метеоусловий для утверждённого маршрута;
   - при нахождении на месте запуска выполняет коррекцию прогноза 
     по фактическим данным (раздел 8.4.3).

б) Оператор БВС заполняет Акт оценки метеорологических условий 
   перед выполнением полёта (форма АМУ-БВС-01, Приложение Б).

в) Руководитель полётов проверяет заполненный акт и принимает 
   решение о допуске к полёту.

8.4.1.3 Этап выполнения полёта:
а) Перед каждым вылетом оператор БВС:
   - проверяет текущие метеоусловия визуально;
   - сверяет фактические условия с данными системы MIRA;
   - при значительных расхождениях (>20%) применяет коррекцию.

б) В процессе полёта:
   - осуществляется визуальный контроль метеоусловий;
   - при ухудшении условий ниже МДМУ полёт немедленно прекращается.

8.4.1.4 Этап после полёта:
а) Оператор БВС фиксирует фактические метеоусловия:
   - вносит данные в Журнал учёта метеорологической информации;
   - при наличии расхождений с прогнозом отмечает характер расхождений.

б) Руководитель полётов:
   - анализирует полноту и качество метеорологического обеспечения;
   - при необходимости инициирует служебное расследование.
```

**Срок:** 3 дня

### 2.11. Раздел 8.4.3: Коррекция прогноза по фактическим данным

**Требуемые изменения:**

**Добавить новый подраздел:**

```
8.4.3 Коррекция прогноза по фактическим данным

8.4.3.1 Коррекция прогноза системы MIRA применяется в следующих случаях:
- оператор БВС находится на месте запуска;
- имеются фактические данные с метеодатчиков или визуальных наблюдений;
- прогнозируемые условия отличаются от фактических более чем на 20%.

8.4.3.2 Порядок применения коррекции:
а) Оператор БВС открывает раздел «Сидя на земле» системы MIRA.

б) Вводит фактические данные:
   - скорость и направление ветра (по анемометру или визуально);
   - температуру воздуха (по термометру);
   - относительную влажность (по гигрометру);
   - видимость (визуально по ориентирам);
   - нижнюю границу облаков (визуально или по лидару);
   - отмечает наличие тумана, осадков, снега.

в) Нажимает кнопку «Применить коррекцию».

г) Система MIRA выполняет:
   - расчёт коэффициентов коррекции (windBias, tempOffset, humidityBias);
   - применение поправок ко всем высотам с затуханием;
   - пересчёт Risk Score и рекомендаций.

д) Оператор БВС изучает обновлённые рекомендации и принимает 
   решение о возможности полёта.

8.4.3.3 Требования к фактическим данным:
- данные должны быть получены не ранее чем за 30 минут до применения коррекции;
- рекомендуется использование сертифицированных метеоприборов;
- при визуальной оценке необходимо указывать метод оценки.

8.4.3.4 После применения коррекции оператор БВС обязан:
- сохранить скриншот или PDF-отчёт с результатами коррекции;
- зафиксировать факт применения коррекции в Журнале учёта;
- указать использованные источники фактических данных.
```

**Срок:** 2 дня

### 2.12. Раздел 8.5: Особенности метеообеспечения при специальных видах работ

**Требуемые изменения:**

**Адаптировать под возможности MIRA:**

```
8.5.1 Аэрофотосъёмка и лазерное сканирование

При выполнении работ, требующих стабильного положения БВС (аэрофотосъёмка, 
лазерное сканирование, инспекция объектов), применяются следующие 
дополнительные требования:

а) Максимальная скорость ветра на высоте полёта:
   - для фотограмметрии: не более 7 м/с;
   - для лазерного сканирования: не более 5 м/с.

б) Видимость: не менее 8 км для обеспечения безопасности.

в) Risk Score: не более 2 для фотограмметрии, не более 1 для сканирования.

г) Система MIRA используется для:
   - выбора оптимального времени работ (анализ Flight Windows);
   - контроля стабильности ветра по времени;
   - оценки рисков по всему маршруту.

8.5.2 Инспекция линейных объектов (ЛЭП, трубопроводы)

При инспекции протяжённых объектов:

а) Система MIRA используется для:
   - построения маршрута вдоль объекта;
   - сегментации маршрута на участки по 10 км;
   - оценки рисков для каждого сегмента.

б) При наличии сегментов с Risk Score > 3:
   - рассматривается возможность изменения маршрута;
   - назначается дополнительное время на прохождение опасного участка;
   - планируется резервное окно для повторного прохода.

8.5.3 Сельскохозяйственные работы

При выполнении авиационно-химических работ:

а) Система MIRA используется для:
   - контроля температуры и влажности (влияют на снос препаратов);
   - оценки риска осадков в течение 6 часов после обработки;
   - выбора времени с минимальным ветром (рано утром/поздно вечером).

б) Дополнительные ограничения:
   - температура: +5...+25°C (оптимально для препаратов);
   - влажность: 40-70% (снижение сноса);
   - ветер: 2-5 м/с (равномерное распределение).
```

### 2.18. Приложение Г: Методика расчёта

**Требуемые изменения:**

**Добавить описание алгоритмов MIRA:**

```
Г.3 Методика расчёта Risk Score в системе MIRA

Г.3.1 Общий алгоритм

Risk Score рассчитывается как сумма баллов по каждому параметру:

Risk Score = Σ(баллы по параметрам), макс. 8 баллов

Г.3.2 Балльная оценка параметров

┌──────────────────┬───────────────┬───────────────┬───────────────┐
│ Параметр         │ 0 баллов      │ 1 балл        │ 2 балла       │
├──────────────────┼───────────────┼───────────────┼───────────────┤
│ Ветер 10м        │ ≤10 м/с       │ 10-15 м/с     │ >15 м/с       │
│ Ветер 500м       │ ≤15 м/с       │ 15-20 м/с     │ >20 м/с       │
│ Осадки           │ ≤1,4 мм/ч     │ 1,4-5 мм/ч    │ >5 мм/ч       │
│ Видимость        │ ≥5 км         │ 2-5 км        │ <2 км         │
│ Обледенение      │ LOW           │ MEDIUM        │ HIGH          │
│ Турбулентность   │ LOW           │ MEDIUM        │ HIGH          │
└──────────────────┴───────────────┴───────────────┴───────────────┘

Г.3.3 Алгоритм коррекции прогноза

1. Расчёт коэффициентов:
   windBias = windActual / windForecast
   tempOffset = tempActual - tempForecast
   humidityBias = humidityActual / humidityForecast

2. Применение к высотам с затуханием:
   weight = exp(-altitude / 1000)
   correctedWind = forecastWind × windBias × weight + forecastWind × (1-weight)
   correctedTemp = forecastTemp + tempOffset × weight

3. Пересчёт Risk Score с использованием скорректированных данных.

Г.3.4 Интерпретация Risk Score

Risk Score 0-2: LOW риск — полёт разрешён
Risk Score 3-5: MEDIUM риск — полёт ограничен
Risk Score 6-8: HIGH риск — полёт запрещён
```
