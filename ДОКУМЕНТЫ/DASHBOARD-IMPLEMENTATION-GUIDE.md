## 🔧 КАК ЭТО РАБОТАЕТ

### 1. Инициализация

```javascript
// При загрузке страницы
DashboardModule.init();
```

### 2. Активация кнопки

Кнопка "ДАШБОРД" активируется при наличии данных:
- Метеоданные (`WeatherModule.cachedData`)
- Сегменты маршрута (`RouteModule.segments`)
- Наблюдения пилота (`PilotObservationsModule`)

### 3. Открытие дашборда

```javascript
// Клик по кнопке
DashboardModule.open();

// Проверка доступности вкладок
DashboardModule.updateTabsAvailability();

// Переключение на первую доступную вкладку
DashboardModule.switchTab('meteo');
```

### 4. Рендер вкладки

```javascript
// 1. Вызов render()
const html = DashboardTabsMeteo.render();

// 2. Вставка в DOM
document.getElementById('dashboardBody').innerHTML = html;

// 3. Пост-обработка (графики)
DashboardTabsMeteo.afterRender();
```

## 📝 ПРИМЕЧАНИЯ

### Почему модальное окно, а не fullscreen CSS?

**ТЗ требует:** модальное окно на весь экран (fixed, z-index: 9999)

**Преимущества:**
- ✅ Изоляция от остального UI
- ✅ Собственная система управления
- ✅ Легче тестировать
- ✅ Не конфликтует с fullscreen правой панели

### Почему 6 вкладок, а не 4 шага?

**Дашборд** — это отдельный поток работы, не связанный с wizard.

**Wizard (3 шага):**
1. Маршрут + Метео
2. Пилот
3. Отчёт

**Дашборд (6 вкладок):**
- Доступен после шага 1
- Объединяет все данные
- Быстрый доступ к любой информации
