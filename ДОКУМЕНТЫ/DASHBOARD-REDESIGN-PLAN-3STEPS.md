### Заменить:

```javascript
const WizardModule = {
    currentStep: 1,
    totalSteps: 4,  // ❌
    // ...
};
```

### На:

```javascript
const WizardModule = {
    currentStep: 1,
    totalSteps: 3,  // ✅
    stepData: {},
    currentTab: 'recommendations',
    callbacks: {},

    init(options = {}) {
        this.currentStep = 1;
        this.stepData = {
            date: Utils.formatDateTimeLocal(Utils.getTomorrowDate()).split('T')[0],
            time: '00:00',
            route: null,

            pilotData: null,       // Вводится на шаге 2
            pilotObservations: [],
            correctedAnalysis: null
        };
        // ...
    }
};
```

## 🎯 ПОЧЕМУ 3 ШАГА ЛУЧШЕ

### Преимущества:

1. **Логическая целостность:**
   - Шаг 1: Получаем данные (маршрут + метео)
   - Шаг 2: Корректируем данные (пилот)
   - Шаг 3: Экспортируем результат (отчёт)

2. **Упрощение UX:**
   - Меньше кликов для достижения цели
   - Нет дублирования (анализ на шаге 2 и в дашборде)
   - Понятная последовательность

3. **Согласованность с ТЗ:**
   - DASHBOARD-PROMPT.md описывает 3 этапа
   - Пользовательский сценарий: 3 шага

4. **Дашборд — отдельный поток:**
   - Дашборд открывается поверх основной страницы
   - Не является частью wizard
   - Доступен после шага 1 (получения метеоданных)

## 🔄 MIGRATION GUIDE

### Для пользователей:

**До:**
```
1. Выбрать дату/время → 2. Построить маршрут → 3. Анализ → 4. Пилот → 5. Отчёт
```

**После:**
```
1. Построить маршрут + Анализ → 2. Пилот → 3. Отчёт
```
